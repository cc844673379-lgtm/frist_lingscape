# 前端规范 (Frontend Specification)

> 本文件对应需求文档 §8、§9、§10。
> 涵盖视觉设计（色彩/字体/Logo/栅格/组件）、动效需求、静态技术需求。
> **设计 Tokens 的 YAML 配置见 [docs/CONFIG.md](./CONFIG.md)。**

---

## 1. 视觉设计（§8）

### 1.1 色彩 Tokens（§8.2，严格使用以下命名与色值，不得改名/改值/缺失）

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--bg` | `#0E1012` | 全站主背景，参考 Mapbox 深色平台底色 |
| `--bg-soft` | `#15171B` | 卡片、能力层、面板背景 |
| `--bg-panel` | `#111815` | 情境地图和治理面板底色，可略带墨绿 |
| `--ink` | `#FFFFFF` | 主文字 |
| `--ink-soft` | `#A0AABA` | 正文和说明文字 |
| `--muted` | `#566171` | 弱提示、辅助标签 |
| `--line` | `#333943` | 分割线、卡片边界、地图网格 |
| `--accent` | `#35D0A9` | 唯一强强调色，按钮、路径、关键节点 |
| `--accent-warm` | `#C8A760` | 低频产业标签、行业分类，不用于主按钮 |
| `--light-band` | `#E8ECE7` | 产品矩阵或可信背书的浅色节奏区 |

**使用原则**：
- 全站只能有一个强行动色：`--accent`。
- `--accent-warm` 只用于行业标签、章节标识、少量图形节点，不能和青绿色同时大面积出现。
- 深色区占页面 70% 左右，浅色区占 30% 以内，用于打断阅读疲劳。
- 禁止使用蓝紫渐变、彩虹渐变、荧光粒子和高饱和多色图标。
- 重要文字与背景对比必须达到可读标准，不能把正文放在低对比纹理或动效上。

### 1.2 字体方向（§8.3）
字体系统以几何/现代无衬线为主，宋体只作为品牌强调，不作为全站标题默认。

**推荐字体角色**：
| 角色 | 字体方向 | 用途 |
| --- | --- | --- |
| Display | 几何无衬线 / 黑体，字重 500-700 | Hero、一级标题、关键数字、区块标题 |
| Body | 现代无衬线，字重 400 | 正文、说明、卡片内容 |
| Mono | 等宽字体 | 英文标签、技术字段、状态、调用链 |
| Brand Serif | 宋体/衬线，少量使用 | 品牌字样、极少量强调短句 |

**建议字号**：
| Token | 桌面端 | 移动端 | 说明 |
| --- | --- | --- | --- |
| Hero | 72-96px | 40-48px | 行高 0.98-1.08，避免过粗 |
| H1 | 56-64px | 34-40px | 一级区块标题 |
| H2 | 36-44px | 28-32px | 二级区块标题 |
| Body large | 20-22px | 17-18px | 首屏副标题和重点说明 |
| Body | 15-17px | 14-16px | 正文和卡片 |
| Label | 11-13px | 11-12px | 英文 overline、技术字段 |

**规则**：
- 标题优先左对齐，避免大面积居中排版。
- 不使用负字距作为通用规则；只有 Hero 超大字号可轻微收紧。
- 中文正文不低于 14px，移动端卡片正文不低于 13px。
- **宋体/衬线只允许用于三处**：①首屏主标题中的"羚境"二字；②品牌理念区的核心判断短句（不超过 20 个汉字）；③页脚 Logo 区品牌名。除此之外，标题、正文、卡片、按钮、导航全部使用几何/现代无衬线。

### 1.3 Logo 与图形母题（§8.4）
Logo/视觉系统应围绕三个母题：
- **抽象羚角**：代表敏锐、洞察、穿越复杂环境。
- **情境边界**：代表真实业务现场、组织边界、系统边界。
- **智能路径**：代表 AI 连接模型、知识、工具、流程与行动。

**图形要求**：
- 避免具象羚羊头像。
- 避免普通字母 L 或 A 的简单变形。
- 标志需适配官网、PPT、产品界面、公众号头像和系统图标。
- 官网首屏动态视觉应能从 Logo 母题延展出来。

### 1.4 栅格与布局（§8.5）
参考 Mapbox 的技术平台式栅格，采用清晰、稳定、可扩展的布局结构。

**布局 tokens**：
- 页面最大宽度：`1280px`
- 桌面端主栅格：12 columns
- 平板端主栅格：8 columns
- 移动端主栅格：4 columns
- 栅格 gutter：`24px`
- 桌面端页面边距：`48px`
- 平板端页面边距：`32px`
- 移动端页面边距：`20px`
- 正文最大宽度：`680px`
- 长说明段落行长：中文 28-36 字为宜

**分区规则**：
- Hero 可全宽，但正文内容必须落在栅格内。
- 解决方案、产品矩阵、治理面板必须与同一栅格对齐。
- 不使用随机漂浮卡片；卡片必须服务某个明确的数据、流程、能力或场景。
- 桌面端解决方案可使用 5 列或 3+2 结构；移动端必须改为单列或横向滑动。

### 1.5 组件规范（§8.6）

**组件 token 映射**：
| 组件 | 背景 | 文字 | 边框/分割 | 强调/hover |
| --- | --- | --- | --- | --- |
| 全站页面 | `--bg` | `--ink` / `--ink-soft` | - | - |
| Hero canvas 容器 | `--bg-panel` | DOM 文案使用 `--ink` | 网格线 `--line` | 路径和节点使用 `--accent` |
| Hero 能力模块卡片 | `--bg-soft` | `--ink` / `--ink-soft` | `1px solid --line` | hover 边框 `--accent` |
| 行业解决方案卡片 | `--bg-soft` | `--ink` / `--ink-soft` | `1px solid --line` | hover 边框 `--accent`，标签可用 `--accent-warm` |
| 能力体系层级 | `--bg-soft` | `--ink` / `--ink-soft` | 左侧路径线 `--line` | 当前层路径线 `--accent` |
| 产品矩阵区 | `--light-band` | 深色文字 `#111714` / `#405049` | `rgba(14,16,18,0.18)` | hover 背景加深，不使用强阴影 |
| 治理面板 | `--bg-panel` | `--ink` / `--ink-soft` | `1px solid --line` | 状态点 `--accent`，警示不使用红色大面积铺底 |
| 可信背书卡片 | 深色区用 `--bg-soft`，浅色区用 `--light-band` | 按背景反差选择 | `--line` 或浅色区深色 18% 边框 | 只做边界/文字高亮 |
| 顶部导航 | 透明到 `rgba(14,16,18,0.82)` | `--ink-soft` | 底边 `--line` | 当前项 `--ink` 或 `--accent` |
| 主按钮 | `--accent` | `--bg` | `1px solid --accent` | hover 降低亮度或轻微上移 |
| 次按钮 | transparent | `--ink` | `1px solid --line` | hover 边框 `--accent` |

**按钮**：
- 主按钮使用 `--accent` 实心或高对比描边。
- 次按钮使用透明背景 + `--line` 边界。
- 按钮圆角使用 `999px`，高度不低于 `44px`。
- 按钮文案必须是动作：查看能力体系、获取方案、预约沟通、联系咨询。

**卡片**：
- 默认背景 `--bg-soft`，边框 `1px solid --line`。
- 圆角控制在 `8px`，不使用大圆角软卡片。
- 卡片 hover 只允许边界高亮、背景轻微变化或路径线亮起，不使用大阴影弹跳。
- 卡片内容结构固定为：标签、标题、说明、能力点/结果点。

**行业入口**：
- 每个行业入口必须包含"行业复杂性、羚境进入方式、典型能力、可交付成果"。
- 不允许只写行业名称和三条功能点。
- 文旅、风控、乡村、企业转型、AI 原生能力的卡片结构必须一致，避免视觉上像散装功能列表。

**深浅节奏**：
- 深色区占 70%，浅色区占 30% 以内。
- `--light-band` 仅用于产品矩阵区和可信背书区。

**绝对禁止**：
- 蓝紫渐变、彩虹渐变、荧光粒子、高饱和多色图标。
- 大面积玻璃拟态卡片。
- 抽象大脑/机器人/芯片/云朵等通用 AI 图标。
- 纯营销落地页式大空白 hero。
- 只有产品功能没有行业入口的 SaaS 官网结构。
- 只有概念口号没有交付路径的 AI 品牌页。

## 2. 动效需求（§9）

### 2.1 动效原则（§9.1）
- 动效必须服务"AI 进入真实产业情境"的叙事。
- 不做无意义漂浮装饰。
- 不因动效导致内容不可见。
- 所有核心内容在 JS 失败时仍应可见。
- 支持 `prefers-reduced-motion`，用户开启减少动态时应降低或关闭动画。

**动效 tokens**：
| Token | 数值 | 用途 |
| --- | --- | --- |
| `--motion-micro` | `150ms` | hover、按钮、导航状态 |
| `--motion-small` | `300ms` | 卡片边界、面板状态 |
| `--motion-medium` | `800ms` | 区块进入、路径线生长 |
| `--ease-mapbox` | `cubic-bezier(0.19, 1, 0.22, 1)` | 主要过渡 |
| `--ease-standard` | `cubic-bezier(0.25, 0.1, 0.25, 1)` | 通用状态变化 |

### 2.2 必须实现的动效（§9.2）
1. **首屏动态情境地图**：边界区域轻微呼吸；智能路径持续流动；节点沿路径移动。优先使用原生 canvas 2D 或 CSS 实现；Three.js 只在确有必要时引入，且必须懒加载，不能阻塞首屏文字和 CTA 渲染。移动端默认使用简化 canvas 或静态路径，不加载 Three.js。
2. **滚动进入动效**：标题、段落、卡片进入视口时轻微上移并淡入。**默认状态应可见；JS 加载后再添加可动画状态**（通过 `html.js-ready` 类启用初始态）。
3. **能力体系层级高亮**：每层进入视口时左侧路径线从下向上生长；鼠标悬停时模块标签亮起。
4. **卡片 hover**：行业解决方案卡片 hover 时边界线变亮；产品卡片 hover 时显示轻微阴影或背景变化。
5. **导航状态**：滚动后顶部导航背景增强；当前区块可选高亮。
6. **品牌理念区**：滚动进入时出现一条细线扫描动效。

### 2.3 动效禁止项（§9.2.1，绝对禁止）
- 不做全屏粒子海。
- 不做复杂三维模型加载作为首屏核心。
- 不做多色霓虹渐变流动。
- 不做大幅视差导致阅读困难。
- 不做滚动劫持。
- 不让内容默认不可见后等待 JS 才显示。
- 不在移动端持续运行高复杂度 canvas 动画。
- 不做长文案打字机逐字显示，尤其不能用于首屏主标题或副标题。
- 不做鼠标跟随光晕、光标拖尾或持续追踪特效；如需鼠标影响路径，只允许影响 canvas 内部路径的极轻微偏移，不能生成独立光标装饰层。

### 2.4 可选增强动效（§9.3，按需实现）
- Logo 开场 1 秒路径绘制。
- 鼠标移动影响首屏路径微弱偏移。
- 页面滚动驱动路径进度。
- 行业卡片 hover 时出现对应业务边界轮廓。

### 2.5 Canvas 技术规范
- Canvas 必须在背景层（z-index < 文本层）。
- DOM 卡片层级高于 canvas。
- Canvas 必须设置 `aria-hidden="true"`。
- 必须监听 `@media (prefers-reduced-motion: reduce)`，开启时停止复杂 Canvas 绘制并取消位移动画。
- 所有核心内容在 JS 失败时仍应可见。
- Canvas 帧率稳定在 60fps，使用 `requestAnimationFrame` + `visibilitychange` 暂停（页面不可见时停止绘制）。

## 3. 静态技术需求（§10）

### 3.1 技术栈（§10.1）
推荐两种方案：
- **方案 A：纯静态 HTML/CSS/JavaScript**：适合快速交付、易部署；适合当前品牌官网第一版；无构建依赖，直接打开 HTML 可预览。
- **方案 B：Vite + 静态构建**：适合后续多页面、组件化和长期维护；构建后仍输出纯静态文件；可更好管理样式、动效和资源。

**本期采用方案 B**：Vite + 静态构建（构建产物输出至 `/dist`，启用 `emptyOutDir: true`）。

**技术约束**：
- 纯静态、无 React/Vue/Angular/Svelte 框架。
- 禁止引入服务端依赖。
- 优先原生能力（Canvas API、IntersectionObserver、Clipboard API）。
- Three.js 仅在确有必要时引入，必须懒加载，不进入首屏关键渲染路径。

### 3.2 资源要求（§10.2）
- 所有图片、字体、脚本应本地化或使用稳定 CDN。
- 首屏关键内容不依赖远程接口。
- 图片使用 WebP/AVIF 优先，保留 PNG/JPG 兼容。
- 静态资源按目录组织：
  - `/assets/images/`
  - `/assets/icons/`
  - `/assets/fonts/`
  - `/assets/scripts/`
  - `/assets/styles/`

### 3.3 性能要求（§10.3）
- 首页首屏资源应控制在合理范围，避免大型视频作为默认背景。
- 首屏动态 canvas 不应造成明显掉帧。
- 首屏动画优先使用 CSS + 轻量 canvas；Three.js 只作为增强层懒加载，不能进入首屏关键渲染路径。
- 移动端应减少粒子数量和动画复杂度。
- 图片需压缩。
- **Lighthouse Performance 建议不低于 85**。

### 3.4 兼容性要求（§10.4）
**浏览器**：
- Chrome 最新版
- Safari 最新版
- Edge 最新版
- 移动端 Safari
- Android Chrome

**屏幕**：
- 1440px 桌面
- 1280px 笔记本
- 768px 平板
- 390px 移动端

所有屏幕均不得出现横向溢出。
