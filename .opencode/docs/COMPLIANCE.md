# 验收标准与清单 (Compliance & Acceptance)

> 本文件对应需求文档 §12。
> 涵盖内容验收、视觉验收、动效验收、技术验收四类标准与可执行清单。
> QA_Compliance_Agent 必须按本文件逐项校验。

---

## 1. 验收标准（§12）

### 1.1 内容验收（§12.1）
- 首屏能清楚表达品牌名、主张和业务方向。
- 页面能解释"产业情境智能"的含义。
- 五类解决方案完整出现。
- RelayAI、ShadowMind、企业知识库完整出现。
- 交付方法、安全治理、行业可信背书和联系咨询完整出现。
- 没有真实案例时，不得出现虚构客户、虚构数据或虚构合作关系。
- 页面不再出现"概念稿""draft"等对外不适合文案。

### 1.2 视觉验收（§12.2）
- 第一屏具备明确科技感和品牌识别。
- 动态视觉与"抽象羚角、情境边界、智能路径"有关。
- 不像通用 AI 模板站。
- 主体内容不像普通 AI 炫技页，应具备成熟产业解决方案官网的信息密度和决策路径。
- 信息架构、栅格和技术平台感应接近 Mapbox 的克制清晰，而不是普通营销落地页。
- 商务 CTA、交付方法和咨询区应接近 Atlantistech 的直接、权威、行动导向。
- 产业质感、可信背书和成果区应接近 Icam Inox 的强排版、清晰边界和少装饰。
- 暗色区与浅色区节奏清楚。
- 桌面端和移动端都无文字重叠、压边和横向滚动。

### 1.3 动效验收（§12.3）
- 首屏 canvas 正常绘制。
- 滚动动效不卡顿。
- JS 失败时核心文字仍可见。
- 开启减少动态偏好时不出现强动画。
- 移动端动画复杂度降低。
- 动效时间和缓动符合 `150ms / 300ms / 800ms` 与 `cubic-bezier(0.19, 1, 0.22, 1)` 的克制节奏。
- 页面没有全屏粒子海、滚动劫持、复杂 3D 首屏加载或多色霓虹渐变。

### 1.4 技术验收（§12.4）
- 可直接静态部署。
- 无控制台 JavaScript 报错。
- 所有导航锚点有效。
- 页面无横向溢出。
- 图片路径和静态资源路径正确。
- **Lighthouse Performance 建议不低于 85，Accessibility 建议不低于 90**。

## 2. 验收清单（可执行 Checklist）

### 2.1 内容验收
- [ ] 首屏能清楚表达品牌名、主张和业务方向
- [ ] 页面能解释"产业情境智能"的含义
- [ ] 五类解决方案完整出现（文旅/金融风控/数字乡村/企业数字化转型/AI 原生能力建设）
- [ ] RelayAI、ShadowMind、企业知识库完整出现
- [ ] 交付方法、安全治理、行业可信背书和联系咨询完整出现
- [ ] 没有虚构客户、数据或合作关系
- [ ] 页面无"概念稿""draft"等对外不适合文案
- [ ] 公司主体"湖北奥羚技术有限公司"在关于公司区出现
- [ ] 能力体系为三层（应用层/情境层/底座层），未改为四层
- [ ] 交付方法为六步流程，未改为成果清单
- [ ] 安全与治理为独立区块，未与交付方法合并
- [ ] 首屏包含 4 个业务证据模块（模型网关/企业知识库/流程连接/行业助手）

### 2.2 视觉验收
- [ ] 第一屏具备明确科技感和品牌识别
- [ ] 动态视觉与"抽象羚角、情境边界、智能路径"有关
- [ ] 不像通用 AI 模板站
- [ ] 信息架构、栅格和技术平台感应接近 Mapbox 的克制清晰
- [ ] 商务 CTA、交付方法和咨询区应接近 Atlantistech 的直接、权威、行动导向
- [ ] 产业质感、可信背书和成果区应接近 Icam Inox 的强排版、清晰边界、少装饰
- [ ] 暗色区与浅色区节奏清楚（深色 70% / 浅色 30% 以内）
- [ ] 桌面端和移动端都无文字重叠、压边和横向滚动
- [ ] 按钮圆角 `999px`，高度不低于 `44px`
- [ ] 卡片圆角 `8px`，边框 `1px solid --line`
- [ ] 全站只有一个强强调色 `--accent: #35D0A9`
- [ ] `--accent-warm: #C8A760` 仅用于低频产业标签
- [ ] 不出现蓝紫渐变、玻璃拟态、发光大脑/机器人/芯片/云朵等通用 AI 图标
- [ ] 宋体/衬线仅用于三处（首屏"羚境"二字、品牌理念核心短句、页脚品牌名）

### 2.3 动效验收
- [ ] 首屏 canvas 正常绘制
- [ ] 滚动动效不卡顿
- [ ] JS 失败时核心文字仍可见
- [ ] 开启减少动态偏好时不出现强动画
- [ ] 移动端动画复杂度降低
- [ ] 动效时间符合 150ms / 300ms / 800ms
- [ ] 缓动符合 `cubic-bezier(0.19, 1, 0.22, 1)` 与 `cubic-bezier(0.25, 0.1, 0.25, 1)`
- [ ] 页面无全屏粒子海、滚动劫持、复杂 3D 首屏加载、多色霓虹渐变、鼠标跟随光晕
- [ ] Canvas 帧率稳定 60fps
- [ ] 页面不可见时 canvas 停止绘制（visibilitychange）

### 2.4 技术验收
- [ ] 可直接静态部署，无服务端依赖
- [ ] 无控制台 JavaScript 报错
- [ ] 所有导航锚点有效
- [ ] 页面无横向溢出
- [ ] 图片路径和静态资源路径正确
- [ ] 静态资源按目录组织（`/assets/images`、`/assets/icons`、`/assets/fonts`、`/assets/scripts`、`/assets/styles`）
- [ ] 首屏关键内容不依赖远程接口
- [ ] 图片使用 WebP/AVIF 优先，保留 PNG/JPG 兼容
- [ ] Lighthouse Performance ≥ 85，Accessibility ≥ 90
- [ ] 浏览器兼容：Chrome/Safari/Edge 最新版、移动端 Safari、Android Chrome
- [ ] 屏幕兼容：1440px 桌面、1280px 笔记本、768px 平板、390px 移动端，均无横向溢出

### 2.5 SEO 与 A11y 验收
- [ ] `<title>` 为"羚境 Lingscape - 让 AI 进入真实产业情境"
- [ ] `<meta name="description">` 存在
- [ ] Open Graph 标题、描述、图片存在
- [ ] 页面语种 `zh-CN`
- [ ] 关键词覆盖完整
- [ ] H1/H2/H3 层级无跳跃
- [ ] 文字与背景对比度达标
- [ ] 所有按钮/链接可键盘聚焦，`:focus-visible` 有清晰焦点指示
- [ ] 首屏 Canvas 带有 `aria-hidden="true"`
- [ ] 关键内容不只存在于图片或 canvas
- [ ] 移动端按钮触控区域不低于 44px
- [ ] 支持 `prefers-reduced-motion`

> SEO 与 A11y 详细规范见 [docs/SEO_A11Y.md](./SEO_A11Y.md)。
